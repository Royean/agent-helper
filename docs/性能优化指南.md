## 🔧 已实施优化

### 1. 连接优化

#### 断线自动重连
```python
# 客户端重连逻辑（指数退避）
async def reconnect_with_backoff():
    max_retries = 10
    base_delay = 1.0
    
    for attempt in range(max_retries):
        delay = min(base_delay * (2 ** attempt), 60)  # 最多 60 秒
        
        try:
            await connect()
            return True
        except Exception as e:
            print(f"重连失败 ({attempt+1}/{max_retries}), {delay}s 后重试")
            await asyncio.sleep(delay)
    
    return False
```

#### 连接池管理
- 复用 WebSocket 连接
- 避免频繁创建/销毁连接
- 限制最大连接数

### 2. 心跳优化

#### 智能心跳
```python
# 动态调整心跳间隔
HEARTBEAT_BASE = 30  # 基础间隔 30 秒

def adjust_heartbeat(success_rate: float):
    """根据连接质量调整心跳间隔"""
    if success_rate > 0.95:
        return min(HEARTBEAT_BASE * 2, 120)  # 质量好，延长间隔
    elif success_rate < 0.8:
        return max(HEARTBEAT_BASE / 2, 10)   # 质量差，缩短间隔
    return HEARTBEAT_BASE
```

#### 心跳压缩
- 心跳包使用最小 JSON
- 减少网络流量

### 3. 消息压缩

#### 大数据传输
```python
import gzip
import json

def compress_message(data: dict) -> bytes:
    """压缩大消息"""
    json_str = json.dumps(data)
    if len(json_str) > 1024:  # 大于 1KB 才压缩
        return gzip.compress(json_str.encode())
    return json_str.encode()

def decompress_message(data: bytes) -> dict:
    """解压缩消息"""
    try:
        return json.loads(gzip.decompress(data))
    except:
        return json.loads(data.decode())
```

```python
@app.middleware("http")
async def performance_monitor(request, call_next):
    start_time = time.time()
    response = await call_next(request)
    duration = time.time() - start_time
    
    # 记录慢请求
    if duration > 1.0:
        print(f"慢请求：{request.url.path} - {duration:.2f}s")
    
    return response
```

## 🔍 故障排查

### 常见问题

#### 1. 连接频繁断开
```bash
# 检查网络质量
ping -c 100 server.example.com

# 查看丢包率
mtr server.example.com
```

**解决方案：**
- 增加心跳间隔
- 启用断线重连
- 检查防火墙设置

#### 2. 内存占用过高
```bash
# 查看内存使用
ps aux | grep agentlinker

# 分析内存泄漏
python3 -m tracemalloc
```

**解决方案：**
- 限制日志缓冲区
- 及时清理过期数据
- 使用内存分析工具

#### 3. CPU 占用过高
```bash
# 查看 CPU 使用
top -p $(pgrep -f agentlinker)

# 分析性能瓶颈
python3 -m cProfile -o profile.stats client.py
```

**解决方案：**
- 优化热点代码
- 减少序列化开销
- 使用异步 I/O
